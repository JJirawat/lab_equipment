# Lab Equipment Management App (แอปพลิเคชันจัดการยืม-คืนอุปกรณ์ห้องแล็บ)

โปรเจคนี้เป็นแอปพลิเคชันสำหรับบริหารจัดการการยืม-คืนอุปกรณ์ในห้องปฏิบัติการ (Lab Equipment) พัฒนาด้วยเฟรมเวิร์ก **Flutter** รองรับการทำงานข้ามแพลตฟอร์ม และใช้ **Firebase** เป็นระบบ Backend เต็มรูปแบบ ทั้งในส่วนของการจัดการฐานข้อมูล (Database) และการยืนยันตัวตนผู้ใช้งาน (Authentication)

---

## 🛠 เทคโนโลยีและเครื่องมือที่ใช้ (Tech Stack)

* **Framework:** Flutter SDK (รองรับเวอร์ชัน `>=3.12.2`)
* **Language:** Dart
* **Backend:** Firebase (ตั้งค่าผ่านไฟล์ `firebase_options.dart`)

### Dependencies หลักที่ใช้งาน (`pubspec.yaml`)
* `firebase_core` (`^4.11.0`): โครงสร้างพื้นฐานสำหรับการเชื่อมต่อแอปพลิเคชันกับบริการต่างๆ ของ Firebase
* `firebase_auth` (`^6.5.4`): ระบบยืนยันตัวตน (Authentication) เพื่อจัดการการเข้าสู่ระบบ
* `cloud_firestore` (`^6.6.0`): ฐานข้อมูลแบบ NoSQL เพื่อจัดเก็บข้อมูลอุปกรณ์, ประวัติการยืม-คืน, ข้อมูลผู้ใช้ และการแจ้งเตือน
* `intl` (`^0.20.3`): สำหรับจัดการรูปแบบ (Formatting) ของข้อมูล เช่น วันที่และเวลา ให้แสดงผลในรูปแบบสากลและเข้าใจง่าย
* `cupertino_icons` (`^1.0.8`): ชุดไอคอนสำหรับตกแต่ง UI

## 📊 โครงสร้างฐานข้อมูล (Firestore Collections)

ระบบถูกออกแบบให้ใช้ Cloud Firestore โดยแบ่งคอลเลกชันหลักๆ ดังนี้:

1. **`users`**: เก็บข้อมูลผู้ใช้งาน
   * ข้อมูลที่เก็บ: ชื่อ (`name`), สิทธิ์การใช้งาน (`role` เช่น `student` หรือ `teacher`)
2. **`equipment`**: เก็บข้อมูลอุปกรณ์ห้องปฏิบัติการ
   * ข้อมูลที่เก็บ: ชื่ออุปกรณ์ (`name`), จำนวนที่เหลืออยู่ (`availableQty`), การมีขนาดอุปกรณ์ (`hasSizes`), รายละเอียดแต่ละขนาด (`sizes` เช่น size, availableQty), วันที่เพิ่มข้อมูล (`createdAt`)
3. **`borrow_requests`**: เก็บประวัติและรายการขอยืมอุปกรณ์
   * ข้อมูลที่เก็บ: รหัสผู้ยืม (`studentId`), ชื่อผู้ยืม (`studentName`), อุปกรณ์ที่ยืม (`equipmentId`, `equipmentName`), ขนาด (`size`), จำนวน (`quantity`), วันที่ยืม (`borrowDate`), วันที่กำหนดคืน (`returnDate`), วัตถุประสงค์ (`purpose`), สถานะการยืม (`status` เช่น `borrowed`, `returned`), วันที่ทำรายการ (`createdAt`)
4. **`notifications`**: เก็บข้อมูลการแจ้งเตือนต่างๆ ภายในระบบ
   * ข้อมูลที่เก็บ: ประเภทการแจ้งเตือน (`type` เช่น `borrow`), รายละเอียดการแจ้งเตือน, สถานะการอ่าน (`isRead`), วันที่แจ้งเตือน (`createdAt`)

## 📱 โครงสร้างหน้าจอการทำงาน (Screens Flow & Architecture)

โค้ดหน้าจอทั้งหมดจะอยู่ในโฟลเดอร์ `lib/screens/` โดยถูกแบ่งลอจิกการทำงานตาม **"บทบาท (Role)"** ของผู้ใช้ 2 กลุ่มหลัก คือ **นักเรียน (Student)** และ **อาจารย์/ผู้ดูแล (Teacher)**

### 1. ระบบยืนยันตัวตน (Authentication)
* **`login_screen.dart`**: หน้าเข้าสู่ระบบ 
  * *การทำงาน:* ผู้ใช้กรอก "รหัสนักเรียน/รหัสครู" และรหัสผ่าน ระบบจะแปลงรหัสเป็นอีเมลจำลอง (เช่น `รหัส@labapp.local`) เพื่อล็อกอินผ่าน Firebase Auth จากนั้นจะเช็คฟิลด์ `role` จากคอลเลกชัน `users` เพื่อพาไปยัง Dashboard ที่ถูกต้อง
* **`register_screen.dart`**: หน้าสมัครสมาชิกใหม่สำหรับนักเรียนและครู

### 2. ส่วนของนักเรียน (Student Flow)
* **`student_dashboard.dart`**: หน้าจอหลักของนักเรียน ประกอบไปด้วย:
  * **Summary Card**: ดึงข้อมูลแบบ Real-time แสดงจำนวนอุปกรณ์ที่ "กำลังยืมอยู่" และคำนวณวันกำหนดคืนเพื่อแสดงอุปกรณ์ที่ "ใกล้ครบกำหนด" (ภายใน 3 วัน)
  * **Quick Menu**: ทางลัดไปดูรายการอุปกรณ์ทั้งหมดและรายการยืมของตัวเอง
  * **New Equipments**: ดึงข้อมูลอุปกรณ์ล่าสุด 8 รายการมาแสดงแบบเลื่อนแนวนอน
  * **Recent Activities**: แสดงประวัติการทำรายการยืม-คืน 5 รายการล่าสุด
* **`equipment_list_screen.dart` (isTeacher: false)**: แสดงรายการอุปกรณ์ทั้งหมด นักเรียนสามารถกดเข้าไปดูรายละเอียดและทำรายการยืมได้
* **`borrow_screen.dart`**: หน้าสำหรับทำรายการยืม
  * *การทำงาน:* มีระบบเลือกขนาด (ถ้ามี), เลือกจำนวน (ห้ามเกินจำนวนคงเหลือ), เลือกวันยืม-วันคืน ระบบจะใช้ **Firestore Transaction** เพื่อรับประกันความถูกต้องของข้อมูล โดยจะทำการลดจำนวนอุปกรณ์ (`availableQty`), สร้าง Document ใน `borrow_requests` และสร้างแจ้งเตือนใน `notifications` พร้อมๆ กัน
* **`my_borrowed_screen.dart`**: ดูรายการที่ตนเองกำลังยืมอยู่และประวัติการคืนอุปกรณ์

### 3. ส่วนของอาจารย์ (Teacher Flow)
* **`teacher_dashboard.dart`**: หน้าจอหลักของอาจารย์
  * มีระบบ Notification Badge บน AppBar ดึงข้อมูลแบบ Real-time ว่ามีคำขอที่ยังไม่ได้อ่าน (`isRead: false`) จำนวนเท่าใด
* **`add_equipment_screen.dart`**: หน้าสำหรับเพิ่มอุปกรณ์ชิ้นใหม่เข้าระบบ สามารถตั้งค่าได้ว่าอุปกรณ์นั้นมีแบ่งตามขนาด (Sizes) หรือไม่
* **`equipment_list_screen.dart` (isTeacher: true)**: หน้าดูรายการอุปกรณ์ในมุมมองของอาจารย์ (อาจมีปุ่มสำหรับแก้ไข/ลบข้อมูล)
* **`notifications_screen.dart`**: หน้าสำหรับดูการแจ้งเตือนเวลามีนักเรียนทำรายการยืม-คืนอุปกรณ์

## 🚀 การเริ่มต้นรันโปรเจค (Getting Started)

1. **เตรียมความพร้อม:** ตรวจสอบให้แน่ใจว่าเครื่องของคุณได้ติดตั้ง **Flutter SDK** (ควรรองรับเวอร์ชัน 3.12.2 ขึ้นไป)
2. **ติดตั้งแพ็กเกจ (Dependencies):** เปิด Terminal เข้าไปที่ Root ของโปรเจคและรันคำสั่ง:
   ```bash
   flutter pub get
   ```
3. **การตั้งค่า Firebase (ถ้าจำเป็นต้องตั้งค่าใหม่):**
   * โปรเจคนี้ได้ผูกกับ Firebase ไว้แล้วผ่านไฟล์ `firebase_options.dart` หากต้องการเปลี่ยนโปรเจค Firebase ให้ใช้เครื่องมือ `flutterfire configure`
4. **รันแอปพลิเคชัน:** 
   * เปิด Emulator (Android/iOS) หรือเสียบสายเชื่อมต่อกับอุปกรณ์จริง
   * รันคำสั่ง:
   ```bash
   flutter run
   ```
